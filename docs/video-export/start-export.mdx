## Overview

Once your video generation is complete, use the export endpoint to create a downloadable MP4 file. This guide covers how to start the export process and handle different scenarios.

## Prerequisites

Before starting an export, ensure:

<Check>Video generation is complete (`status: "completed"`)</Check>
<Check>You have the `video_id` from the generation result</Check>
<Check>The video belongs to your space/account</Check>

## Basic Export

The simplest export request only requires the video ID:

```python
import requests

# Start export with default settings
response = requests.post('https://api.hoox.video/v1/export/start', 
  headers={'Authorization': 'Bearer your_api_key'},
  json={'video_id': 'vid_xyz789'}
)

export_job = response.json()
print(f"Export started: {export_job['job_id']}")
```

This will export the video in its original format.

## Format Conversion

You can convert the video to a different format during export:

```python
# Export as vertical video (9:16)
response = requests.post('https://api.hoox.video/v1/export/start', 
  headers={'Authorization': 'Bearer your_api_key'},
  json={
    'video_id': 'vid_xyz789',
    'format': 'vertical'
  }
)
```

## Webhook Integration

Add a webhook URL to receive automatic notifications when the export completes:

```python
# Export with webhook notification
response = requests.post('https://api.hoox.video/v1/export/start', 
  headers={'Authorization': 'Bearer your_api_key'},
  json={
    'video_id': 'vid_xyz789',
    'format': 'vertical',
    'webhook_url': 'https://your-app.com/webhooks/export-complete'
  }
)
```

Your webhook endpoint will receive a POST request when the export finishes:

```json
{
  "event": "export.completed",
  "job_id": "export_abcd1234",
  "video_id": "vid_xyz789",
  "status": "completed",
  "video_url": "https://storage.hoox.video/exports/export_abcd1234.mp4",
  "timestamp": "2024-01-15T10:35:00Z"
}
```

## Response Handling

The export start endpoint returns a job identifier that you'll use to track progress:

```python
response = requests.post('https://api.hoox.video/v1/export/start', 
  headers={'Authorization': 'Bearer your_api_key'},
  json={'video_id': 'vid_xyz789'}
)

if response.status_code == 201:
    data = response.json()
    job_id = data['job_id']
    status = data['status']  # Always "pending" initially
    estimated_credits = data['estimated_credits']  # Usually 0 for API videos
    
    print(f"Export job {job_id} started")
    print(f"Estimated cost: {estimated_credits} credits")
else:
    print(f"Export failed: {response.json()}")
```

## Common Use Cases

### 1. Multi-Platform Distribution

Export the same video in multiple formats for different platforms:

```python
def export_for_all_platforms(video_id):
    """Export video in all formats for multi-platform distribution"""
    formats = ['vertical', 'square', 'ads']
    jobs = {}
    
    for format_name in formats:
        response = requests.post('https://api.hoox.video/v1/export/start', 
          headers={'Authorization': 'Bearer your_api_key'},
          json={
            'video_id': video_id,
            'format': format_name,
            'webhook_url': f'https://your-app.com/webhooks/export/{format_name}'
          }
        )
        
        if response.status_code == 201:
            jobs[format_name] = response.json()['job_id']
        else:
            print(f"Failed to export {format_name}: {response.json()}")
    
    return jobs

# Usage
jobs = export_for_all_platforms('vid_xyz789')
print(f"Started exports: {jobs}")
```

### 2. Batch Export Processing

Export multiple videos simultaneously:

```python
import asyncio
import aiohttp

async def start_export(session, video_id, format='vertical'):
    """Start export for a single video"""
    async with session.post('https://api.hoox.video/v1/export/start',
                           headers={'Authorization': 'Bearer your_api_key'},
                           json={'video_id': video_id, 'format': format}) as response:
        return await response.json()

async def batch_export(video_ids, format='vertical'):
    """Start exports for multiple videos in parallel"""
    async with aiohttp.ClientSession() as session:
        tasks = [start_export(session, video_id, format) for video_id in video_ids]
        results = await asyncio.gather(*tasks, return_exceptions=True)
        return results

# Usage
video_ids = ['vid_123', 'vid_456', 'vid_789']
results = asyncio.run(batch_export(video_ids, 'vertical'))
```

### 3. Error Handling and Retries

Implement robust error handling for production environments:

```python
import time
import requests
from requests.adapters import HTTPAdapter
from urllib3.util.retry import Retry

def create_robust_session():
    """Create session with automatic retries"""
    session = requests.Session()
    
    retry_strategy = Retry(
        total=3,
        status_forcelist=[429, 500, 502, 503, 504],
        method_whitelist=["HEAD", "GET", "POST"],
        backoff_factor=1
    )
    
    adapter = HTTPAdapter(max_retries=retry_strategy)
    session.mount("http://", adapter)
    session.mount("https://", adapter)
    
    return session

def safe_export_start(video_id, format='vertical', max_retries=3):
    """Start export with comprehensive error handling"""
    session = create_robust_session()
    
    for attempt in range(max_retries):
        try:
            response = session.post(
                'https://api.hoox.video/v1/export/start',
                headers={'Authorization': 'Bearer your_api_key'},
                json={'video_id': video_id, 'format': format},
                timeout=30
            )
            
            response.raise_for_status()
            return response.json()
            
        except requests.exceptions.RequestException as e:
            print(f"Attempt {attempt + 1} failed: {e}")
            if attempt < max_retries - 1:
                time.sleep(2 ** attempt)  # Exponential backoff
            else:
                raise e
```

## Error Codes and Solutions

| Error Code | Description | Solution |
|------------|-------------|----------|
| `MISSING_VIDEO_ID` | No video_id provided | Include video_id in request body |
| `VIDEO_NOT_FOUND` | Video doesn't exist | Verify the video_id is correct |
| `UNAUTHORIZED_VIDEO` | Video not owned by your space | Use a video from your account |
| `VIDEO_NOT_READY` | Generation not complete | Wait for generation to finish |
| `INVALID_FORMAT` | Invalid format specified | Use 'vertical', 'square', or 'ads' |
| `INSUFFICIENT_CREDITS` | Not enough credits | Add credits to your account |

## Rate Limiting

Export requests are subject to rate limiting:

- **Rate Limit**: Based on your API key configuration
- **Headers**: Check `X-RateLimit-Remaining` and `X-RateLimit-Reset`
- **Best Practice**: Space out export requests and use exponential backoff

```python
def check_rate_limit(response):
    """Check rate limit headers and wait if necessary"""
    remaining = int(response.headers.get('X-RateLimit-Remaining', 100))
    reset_time = int(response.headers.get('X-RateLimit-Reset', 0))
    
    if remaining < 5:  # Low on requests
        wait_time = reset_time - int(time.time())
        if wait_time > 0:
            print(f"Rate limit low, waiting {wait_time} seconds...")
            time.sleep(wait_time)
```
