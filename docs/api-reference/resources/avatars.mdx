## Overview

This endpoint returns all available avatar looks that you can use in video generation. Avatar looks are specific appearances/styles of digital presenters that can deliver your video content.

## Authentication

<Info>
This endpoint requires API key authentication. Include your API key in the `Authorization` header.
</Info>

## Query Parameters

<ParamField query="gender" type="string">
  Filter avatars by gender: `male` or `female`.
</ParamField>

<ParamField query="tags" type="string">
  Comma-separated list of tags to filter by (e.g., "business,professional").
</ParamField>

## Response

<ResponseField name="avatars" type="array">
  Array of available avatar look objects:
  <Expandable title="Avatar Look Object">
    <ResponseField name="id" type="string">
      Unique identifier for the avatar look (use this in generation requests).
    </ResponseField>
    <ResponseField name="name" type="string">
      Human-readable name of the avatar look.
    </ResponseField>
    <ResponseField name="gender" type="string">
      Avatar gender: `male` or `female`.
    </ResponseField>
    <ResponseField name="place" type="string">
      Setting/background of the avatar (e.g., "office", "studio", "outdoor").
    </ResponseField>
    <ResponseField name="format" type="string">
      Optimal video format for this avatar (e.g., "vertical", "square").
    </ResponseField>
    <ResponseField name="tags" type="array">
      Array of descriptive tags (e.g., ["business", "professional", "modern"]).
    </ResponseField>
    <ResponseField name="thumbnail" type="string">
      URL to a static thumbnail image of the avatar.
    </ResponseField>
    <ResponseField name="preview" type="string">
      URL to a short video preview of the avatar in action.
    </ResponseField>
  </Expandable>
</ResponseField>

## Example

<RequestExample>

```bash cURL
curl -X GET "https://api.hoox.video/v1/resources/avatars" \
  -H "Authorization: Bearer your_api_key"

# With filters
curl -X GET "https://api.hoox.video/v1/resources/avatars?gender=female&tags=business" \
  -H "Authorization: Bearer your_api_key"
```

```python Python
import requests

url = "https://api.hoox.video/v1/resources/avatars"
headers = {
    "Authorization": "Bearer your_api_key"
}

# Get all avatars
response = requests.get(url, headers=headers)
avatars = response.json()['avatars']

# Filter for business avatars
params = {
    "gender": "female",
    "tags": "business,professional"
}
response = requests.get(url, headers=headers, params=params)
business_avatars = response.json()['avatars']

print(f"Found {len(business_avatars)} business avatars")
for avatar in business_avatars:
    print(f"- {avatar['name']} ({avatar['id']}) - {avatar['place']}")
```

## Format Compatibility

Avatars are optimized for specific video formats:

| Format | Aspect Ratio | Best For | Avatar Positioning |
|--------|-------------|----------|-------------------|
| Vertical | 9:16 | TikTok, Instagram Stories | Upper body focus |
| Square | 1:1 | Instagram posts, LinkedIn | Balanced framing |
| Landscape | 16:9 | YouTube, presentations | Full scene view |

<Tip>
While avatars have optimal formats, they can be used in any format. The system will automatically adjust framing and positioning.
</Tip>

## Best Practices

### 1. Avatar Selection
- **Match avatar to content type**: Professional for business, casual for social
- **Consider your audience**: Age, demographics, cultural relevance
- **Brand consistency**: Use similar avatar styles across content series

### 2. Format Optimization
```python
def select_optimal_avatar(content_type, target_format):
    """Select avatar optimized for specific format"""
    params = {"format": target_format}
    
    if content_type == "business":
        params["tags"] = "business,professional"
    elif content_type == "social":
        params["tags"] = "casual,friendly"
    elif content_type == "educational":
        params["tags"] = "educational,clear"
    
    response = requests.get(f"{API_BASE}/resources/avatars", params=params)
    avatars = response.json()['avatars']
    
    return avatars[0]['id'] if avatars else None
```

### 3. Performance Tips
- **Cache avatar data**: Store avatar information locally
- **Preload previews**: Load thumbnails and previews for better UX
- **Batch requests**: Get both voices and avatars in parallel

```javascript
// Parallel loading of resources
const [voicesResponse, avatarsResponse] = await Promise.all([
  fetch('/api/v1/resources/voices', { headers }),
  fetch('/api/v1/resources/avatars', { headers })
]);

const voices = await voicesResponse.json();
const avatars = await avatarsResponse.json();
```

## Custom Avatar Creation

For Pro and Enterprise plans, you can create custom avatars:

### Requirements
- High-quality video samples (minimum 2 minutes)
- Consistent lighting and background
- Clear audio synchronization
- Multiple poses and expressions

### Process
1. **Upload samples** via dashboard or API
2. **AI training** (typically 24-48 hours)
3. **Quality review** and approval
4. **Avatar availability** in resources list

## Error Codes

- `UNAUTHORIZED`: Invalid or missing API key
- `RATE_LIMIT_EXCEEDED`: Too many requests
- `INTERNAL_ERROR`: Server error retrieving avatars

## Integration Examples

### Avatar Selector Component
```javascript
function AvatarSelector({ onSelect }) {
  const [avatars, setAvatars] = useState([]);
  const [filter, setFilter] = useState({ gender: '', tags: '' });

  useEffect(() => {
    loadAvatars(filter);
  }, [filter]);

  const loadAvatars = async (filters) => {
    const params = new URLSearchParams(filters);
    const response = await fetch(`/api/v1/resources/avatars?${params}`);
    const data = await response.json();
    setAvatars(data.avatars);
  };

  return (
    <div className="avatar-selector">
      <div className="filters">
        <select onChange={(e) => setFilter({...filter, gender: e.target.value})}>
          <option value="">All Genders</option>
          <option value="male">Male</option>
          <option value="female">Female</option>
        </select>
      </div>
      
      <div className="avatar-grid">
        {avatars.map(avatar => (
          <div key={avatar.id} className="avatar-card" onClick={() => onSelect(avatar)}>
            <img src={avatar.thumbnail} alt={avatar.name} />
            <h3>{avatar.name}</h3>
            <p>{avatar.place}</p>
            <div className="tags">
              {avatar.tags.map(tag => <span key={tag}>{tag}</span>)}
            </div>
          </div>
        ))}
      </div>
    </div>
  );
}
```
